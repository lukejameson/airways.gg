💡 Why & How

airways.gg has no programmatic API — all data flows through SvelteKit load functions. When an agent needs to debug a data issue (wrong flight status, missing weather, scraper gap), there is no way to query the database except a direct Postgres connection. The debug API adds a Bearer-token-gated /api/debug/* route group with structured table endpoints, UI-parity mirrors that reuse existing query functions, and a SELECT-only raw SQL escape hatch. Auth is a ~10-line addition to hooks.server.ts checking a DEBUG_API_TOKEN env var against the Authorization header. Users and sessions tables are excluded to avoid exposing password hashes and session tokens.

🎁 Expected Outcome

An agent can GET /api/debug/flights?date=2026-06-07&status=delayed&limit=50 to query flights, GET /api/debug/ui/homepage to get the exact data the homepage renders, GET /api/debug/ui/stats?range=30 for stats dashboard data, and POST /api/debug/sql with {"sql": "SELECT ..."} for ad-hoc queries. All endpoints return JSON with row counts and query timing. All are gated behind Authorization: Bearer <DEBUG_API_TOKEN>. Raw SQL is SELECT-only.
