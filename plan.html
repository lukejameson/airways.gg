Fix systemic 1-hour offset in all flight scheduled departure/arrival times caused by Luxon failing to resolve the Europe/London timezone on Alpine Docker images during BST.
Progress0 / 10

💡 Why & How

Every flight displays scheduled times 1 hour later than reality during BST (e.g., GR634 shows 10:15 instead of 09:15). The root cause is that localToUtc() uses Luxon's DateTime.fromFormat with zone:'Europe/London', but the node:20-alpine Docker images lack tzdata, causing Luxon to fall back to UTC and store times as-if-UTC instead of true UTC. The display layer (formatGuernseyTime) uses native Intl.DateTimeFormat which resolves the timezone correctly, converting the wrongly-stored UTC value back to BST as BST+1 — producing the 1-hour offset.

Fix: install tzdata in guernsey-scraper and web Dockerfiles, run a SQL migration to correct already-stored bad data, and add an integration test to prevent regression.

🎁 Expected Outcome

After deployment, all flight scheduled departure/arrival times display correctly in BST (09:15 instead of 10:15 for GR634). The fix is durable: tzdata installed in affected containers, regression test in place, and existing bad data corrected via migration.
